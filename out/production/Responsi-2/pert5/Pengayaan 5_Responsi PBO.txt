SOAL PENGAYAAN (Refleksi Polimorfisme)
Jawablah pertanyaan berikut:
1. Apa yang dimaksud dengan polimorfisme dalam OOP? Jelaskan bagaimana metode override bekerja.
2. Mengapa array bertipe DefaultMesin dapat menyimpan objek MesinMotor dan MesinTraktor?
3. Apa perbedaan override dan overload?
4. Apa keuntungan menggunakan polimorfisme dalam sistem inventaris seperti contoh di atas?
5. Bagaimana cara kerja dynamic dispatch pada Java saat memanggil method override?
jawaban:

1. Polimorfisme adalah kemampuan satu variabel referensi untuk memanggil method berbeda sesuai objek aslinya.

2. Karena:
-mesinMotor extends defaultMesin
-mesinTraktor extends defaultMesin
Sehingga objek subclass adalah bagian dari superclass (IS-A relationship).

3. Override
-Terjadi di subclass	
-Method sama: nama, parameter, return
-Mengubah perilaku parent
-Implementasi berbeda
Overload
-Terjadi di class yang sama
-Nama sama, parameter beda
-Membuat variasi method
-Method tambahan, bukan pergantian

4. -Bisa menaruh semua mesin dalam satu array
-Tidak perlu membuat method berbeda untuk tiap jenis mesin
-kode lebih sederhana, reusable, dan fleksibel
-Memudahkan penambahan tipe mesin baru tanpa mengubah logika program utama

5. Ketika memanggil method:
defaultMesin m = new mesinMotor(...);
m.nilaiPerforma();
mesinMotor, bukan tipe variabelnya.
Maka method yang dipanggil adalah method override milik mesinMotor.
